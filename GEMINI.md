# GEMINI.md

The development rules for this project live in a single source of truth,
**`AGENTS.md`** at the repo root. Read it and follow it for everything: FVM
usage, architecture/SOLID layering, testing (unit + integration), formatting,
commit/PR conventions, the assignment-before-work rule, and the issue/label
taxonomy.

@AGENTS.md

These rules are intentionally **not** duplicated here. When they change, edit
`AGENTS.md`, not this file.

Quick reminder: run every Flutter/Dart command through FVM
(`fvm flutter analyze`, `fvm flutter test`, `fvm dart format .`) so you build
against the SDK version pinned in `.fvmrc`.
